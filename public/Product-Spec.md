# 家庭消费记账系统 - Product Spec

## 产品定位

一个面向家庭的消费记账web应用，支持多用户共享账本，通过AI技术增强用户体验，实现消费明细录入、分类汇总、数据分析、预算管理等核心功能。

## 目标用户

- 家庭用户，需要记录和管理家庭消费
- 多人共享账本，支持家庭成员协作
- 需要数据分析和预算管理的用户

## 核心功能

### 1. 消费明细录入
- 支持多模态输入：文字、语音、图片
- 支持完整描述和关键词输入
- AI自动识别和分类
- 支持编辑和删除消费记录

### 2. 按月分类汇总
- 自动按月份汇总消费数据
- 支持按分类维度统计
- 支持按成员维度统计

### 3. 数据分析
- 按时间维度查看消费趋势
- 按类别维度查看消费分布
- 按成员维度查看消费占比
- 支持可视化图表展示
- 支持数据表格展示
- 支持导出Excel

### 4. 预算管理
- 设置每月预算
- 查看预算执行情况
- 超支提醒

### 5. 成员对比
- 查看家庭成员的消费占比
- 对比不同成员的消费习惯

### 6. 分类管理
- 系统预设分类体系
- 支持用户自定义分类
- 混合模式：预设+自定义

### 7. 成员管理
- 添加/删除家庭成员
- 设置成员权限
- 查看成员消费明细

## 用户流程

### 记账流程
1. 用户登录系统
2. 进入记账页面
3. 选择输入方式（文字/语音/图片）
4. 输入消费信息（完整描述或关键词）
5. AI自动识别消费信息（金额、类别、时间、备注等）
6. AI自动分类，返回多个可能的分类
7. 用户确认或修改分类
8. 用户确认或修改其他信息
9. 保存消费记录

### 查看统计流程
1. 用户进入统计页面
2. 选择统计维度（时间/类别/成员）
3. 查看可视化图表
4. 查看数据表格
5. 查看AI分析结果（同比分析、异常检测、节省建议、趋势预测）
6. 导出数据（可选）

### 预算管理流程
1. 用户进入预算设置页面
2. 设置每月预算金额
3. 按分类设置预算分配
4. 查看预算执行情况
5. 查看超支提醒

## 应用场景

### 场景1：日常消费记录
用户在超市购物后，打开记账页面，通过语音输入"今天中午吃了30块火锅，和老婆一起"，AI自动识别出金额30、类别餐饮、备注"和老婆一起"，并推荐多个分类"餐饮 > 火锅"、"餐饮 > 外卖"，用户选择"餐饮 > 火锅"并保存。

### 场景2：月度消费分析
用户每月月底进入统计页面，选择本月数据，查看本月消费趋势图表，发现餐饮支出较高，AI分析结果提示"本月餐饮支出比上月增加20%，建议减少外卖频率，多在家做饭"。

### 场景3：预算管理
用户月初进入预算设置页面，设置本月总预算5000元，其中餐饮2000元、交通1000元、购物1000元、其他1000元。月中查看预算执行情况，发现餐饮已超支，系统提醒"餐饮预算已超支200元，请注意控制"。

### 场景4：家庭成员对比
用户进入统计页面，选择成员对比维度，查看家庭成员消费占比，发现妻子的消费占比60%，自己的消费占比40%，进一步查看各成员的消费明细。

## AI增强点

### 1. 智能识别
- **用途**：用户通过自然语言输入消费信息，AI自动解析
- **触发方式**：用户输入文字/语音/图片后自动触发
- **增强效果**：减少用户手动输入，提高记账效率

### 2. AI自动分类
- **用途**：根据消费描述自动推荐分类
- **触发方式**：用户输入消费信息后自动触发
- **增强效果**：减少用户选择分类的时间，提高分类准确性

### 3. 消费分析
- **用途**：AI分析消费数据，提供洞察和建议
- **触发方式**：用户查看统计页面时自动触发
- **增强效果**：帮助用户了解消费习惯，优化消费决策

## AI功能细节

### 智能识别功能
**用途**：解析用户的自然语言输入，提取消费信息

**输入**：
- 文本输入：用户输入的消费描述（完整描述或关键词）
- 语音输入：用户的语音消息（需要先转换为文本）
- 图片输入：消费凭证图片（需要先识别文字）

**输出**：
```json
{
  "amount": 30,
  "category": "餐饮",
  "subcategory": "火锅",
  "date": "2025-01-15",
  "time": "12:30",
  "members": ["用户本人", "妻子"],
  "description": "今天中午吃了30块火锅，和老婆一起",
  "confidence": 0.95
}
```

**约束规则**：
- 金额必须是有效数字
- 日期必须是有效日期格式
- 分类必须在系统预设分类或用户自定义分类中
- 成员必须在现有成员列表中
- 置信度低于0.8时，要求用户确认

### AI自动分类功能
**用途**：根据消费描述推荐可能的分类

**输入**：
- 消费描述文本
- 消费金额
- 现有分类体系

**输出**：
```json
{
  "recommendations": [
    {
      "category": "餐饮",
      "subcategory": "火锅",
      "confidence": 0.9
    },
    {
      "category": "餐饮",
      "subcategory": "外卖",
      "confidence": 0.7
    }
  ]
}
```

**约束规则**：
- 返回2-3个最可能的分类
- 按置信度排序
- 分类必须在现有分类体系中
- 所有推荐分类的置信度之和不超过1

### 消费分析功能
**用途**：分析消费数据，提供洞察和建议

**输入**：
- 消费数据列表
- 时间范围
- 筛选条件（分类、成员）

**输出**：
```json
{
  "monthOverMonth": {
    "currentMonth": 5000,
    "lastMonth": 4500,
    "change": 500,
    "changeRate": 0.111
  },
  "anomalies": [
    {
      "type": "超支",
      "category": "餐饮",
      "amount": 2000,
      "budget": 1500,
      "message": "餐饮支出超出预算500元"
    }
  ],
  "suggestions": [
    "本月餐饮支出比上月增加20%，建议减少外卖频率，多在家做饭",
    "交通支出稳定，继续保持"
  ],
  "prediction": {
    "nextMonth": 5200,
    "trend": "上升"
  }
}
```

**约束规则**：
- 同比分析：对比本月和上月的消费变化
- 异常检测：识别超出平均值30%的消费
- 节省建议：基于消费习惯给出具体建议
- 趋势预测：基于历史数据预测下月消费，准确率不低于70%

## 数据验证规则

### 消费记录验证
- 金额：必须为正数，最多支持小数点后两位
- 日期：必须是有效日期格式（YYYY-MM-DD）
- 时间：必须是有效时间格式（HH:MM）
- 分类：必须在现有分类体系中
- 成员：必须在现有成员列表中
- 备注：最多200个字符

### 用户验证
- 用户名：3-20个字符，只能包含字母、数字、下划线
- 密码：最少8个字符，必须包含字母和数字
- 初始密码：由管理员设置，用户首次登录后必须修改

### 预算验证
- 预算金额：必须为正数，最多支持小数点后两位
- 分类预算分配：总和不能超过总预算

## 错误处理机制

### 错误码定义
```
200 - 成功
400 - 请求参数错误
401 - 未授权（Token无效或过期）
403 - 禁止访问
404 - 资源不存在
409 - 资源冲突（如用户名已存在）
422 - 数据验证失败
500 - 服务器内部错误
```

### 错误提示原则
- 错误信息要清晰具体，帮助用户理解问题
- 避免技术术语，使用用户友好的语言
- 提供解决问题的建议（如果可能）

### 错误处理流程
1. 后端捕获错误
2. 返回标准错误响应（包含错误码和错误信息）
3. 前端根据错误码显示对应的错误提示
4. 记录错误日志（开发环境）

## 安全性细节

### JWT Token安全
- Token有效期：7天
- Token存储：localStorage
- Token刷新：过期后需要重新登录
- Token传递：Authorization Header（Bearer Token）

### XSS防护
- 前端输出时进行HTML转义
- 使用v-html时谨慎处理
- Element Plus组件自带XSS防护

### CSRF防护
- 使用JWT Token进行认证，天然防御CSRF
- 重要操作需要二次确认

### SQL注入防护
- 使用参数化查询
- 使用ORM（Prisma或类似工具）
- 输入验证和过滤

### 密码安全
- 使用bcrypt加密存储
- 密码最少8个字符，必须包含字母和数字
- 不记录明文密码

### API安全
- 所有API需要认证（除了登录和注册）
- 敏感操作需要二次确认
- 限制请求频率（可选）
