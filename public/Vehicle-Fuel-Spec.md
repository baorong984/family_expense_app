# 油费/充电费管理模块需求文档

## 概述

油费/充电费管理模块是家庭消费记账系统的扩展功能，用于记录和管理车辆的燃油费用或电动汽车的充电费用。该模块通过记录每次加油/充电的费用和里程数，自动计算每公里的实际成本，为用户提供车辆使用成本的详细统计和分析。

## 业务背景

在日常生活中，家庭拥有私家车（燃油车或电动车）是常见情况。车主需要：

- **记录每次加油/充电**：包括金额和当前里程数
- **计算成本效率**：通过里程数变化计算每公里的实际成本
- **统计分析支出**：查看一定时间范围内的总花费、平均成本趋势
- **对比不同时期**：了解用车成本的变化情况

当前系统的消费记录功能虽然可以记录交通类支出，但无法：

1. 专门针对车辆加油/充电场景进行结构化记录
2. 自动关联里程数据并计算每公里成本
3. 提供车辆使用的专项统计和分析
4. 追踪每公里成本的变化趋势

因此需要一个专门的模块来管理这部分数据。

## 核心功能

### 1. 车辆信息管理

#### 1.1 车辆基本信息

支持管理多辆车的基本信息：

- **车牌号**：唯一标识（如：京A12345）
- **品牌型号**：如：比亚迪秦PLUS、丰田凯美瑞
- **车辆类型**：燃油车 / 纯电动 / 插电混动
- **初始里程数**：首次记录时的基准里程
- **备注**：可选，如车辆颜色、购买日期等
- **创建时间**
- **修改时间**

#### 1.2 车辆列表

展示所有已添加的车辆，支持：

- 新增车辆
- 编辑车辆信息
- 删除车辆（需确认是否有关联记录）

### 2. 加油/充电记录管理

#### 2.1 加油记录（适用于燃油车、插电混动）

每次加油时记录以下核心信息：

- **所属车辆**：选择哪辆车
- **加油日期**：默认当天
- **加油时间**：可选，精确到小时
- **加油金额**：必填，如 350.00 元
- **当前里程数**：必填，如 12580 公里
- **本次行驶里程**：自动计算 = 当前里程 - 上次里程
- **每公里成本**：自动计算 = 本次金额 ÷ 本次行驶里程
- **备注**：可选
- **创建时间**

#### 2.2 充电记录（适用于纯电动、插电混动）

每次充电时记录以下核心信息：

- **所属车辆**：选择哪辆车
- **充电日期**：默认当天
- **充电时间**：可选，精确到小时
- **充电金额**：必填，如 50.00 元
- **当前里程数**：必填，如 8560 公里
- **本次行驶里程**：自动计算 = 当前里程 - 上次里程
- **每公里成本**：自动计算 = 本次金额 ÷ 本次行驶里程
- **备注**：可选
- **创建时间**

> 💡 **设计理念**：采用极简设计，只记录最核心的两个数据——**花了多少钱**和**跑了多少公里**，系统自动计算每公里成本。无需关心油品类型、加油站、升数/度数等细节信息。

#### 2.3 记录列表

展示所有加油/充电记录，支持：

- **筛选条件**：
  - 按车辆筛选
  - 按类型筛选（加油/充电）
  - 按日期范围筛选
- **排序**：按日期倒序（最新在前）
- **分页**：支持分页显示
- **操作**：编辑、删除单条记录
- **导出**：导出为Excel表格

### 3. 成本计算逻辑

#### 3.1 里程差值计算

```
本次行驶里程 = 当前记录的里程数 - 同一车辆上一条记录的里程数
```

**特殊情况处理**：

- 如果是第一条记录，则"本次行驶里程"显示为 "-"
- "每公里成本"也显示为 "-"
- 如果里程数小于上一条记录（可能重置过里程表），提示用户确认
- 支持手动输入"本次行驶里程"覆盖自动计算值

#### 3.2 单位成本计算

**统一公式（加油和充电通用）**：

```
每公里成本 = 本次金额 ÷ 本次行驶里程（单位：元/km）
```

示例：

- 加油花费 380 元，行驶了 480 公里 → 每公里成本 = 0.79 元/km
- 充电花费 55 元，行驶了 320 公里 → 每公里成本 = 0.17 元/km

#### 3.3 数据验证规则

- 金额必须 > 0
- 里程数必须 >= 0 且 >= 该车辆上次记录的里程数（否则提示警告）
- 如果里程数 < 上次记录，弹出确认框："里程数小于上次记录，是否继续？"

### 4. 统计分析功能

#### 4.1 概览统计卡片

页面顶部展示关键指标：

- **总花费**：选定时间范围内所有加油/充电的总金额
- **总里程**：首条与末条记录的里程差
- **平均每公里成本**：总花费 ÷ 总里程
- **记录次数**：加油/充电次数
- **平均单次花费**：总花费 ÷ 记录次数

按车辆分别统计（如果有多种车）。

#### 4.2 时间维度分析

- **月度统计**：每月的花费、里程、平均成本趋势图
- **年度统计**：每年的总花费对比
- **季度分析**：季度花费分布

**图表展示**：

- 折线图：展示月度花费趋势
- 柱状图：展示各月份花费对比
- 双轴图：同时展示花费和里程

#### 4.3 成本效率分析

- **单位成本趋势**：每公里成本的变化曲线
- **成本波动分析**：识别异常高/低成本的记录

**图表展示**：

- 折线图：每公里成本随时间变化
- 散点图：行驶里程 vs 单次花费（发现异常点）

#### 4.4 对比分析

- **不同车辆对比**：如果有多辆车，对比各车的成本效率
- **同期对比**：本月 vs 上月，今年 vs 去年
- **分类对比**：加油 vs 充电（如果是混动车）

### 5. 数据导出功能

#### 5.1 导出格式

支持导出为 Excel 文件（.xlsx 格式）

#### 5.2 导出内容

可选择导出的字段：

- 基本信息：日期、车辆、类型（加油/充电）、时间
- 费用信息：金额
- 里程信息：当前里程、本次行驶里程、每公里成本
- 其他：备注、创建时间

#### 5.3 导出范围

- 全部记录
- 按日期范围导出
- 按车辆导出
- 按类型导出（仅加油/仅充电）

| 字段名          | 类型     | 长度 | 允许NULL | 默认值                                        | 说明                              |

| record_time     | TIME     | -    | YES      | NULL                                          | 加油/充电时间                     |

| current_mileage | INT      | 11   | NO       | -                                             | 当前里程数（公里）                |

| created_by      | INT      | 11   | NO       | -                                             | 创建者（关联users.id）            |
| created_at      | DATETIME | -    | NO       | CURRENT_TIMESTAMP                             | 创建时间                          |
| updated_at      | DATETIME | -    | NO       | CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间                          |

**索引**：

- PRIMARY KEY: `id`
- INDEX: `vehicle_id`
- INDEX: `record_date`
- INDEX: `record_type`
- INDEX: `created_by`

### vehicles - 车辆表

| 字段名          | 类型     | 长度 | 允许NULL | 默认值                                        | 说明                                                 |
| --------------- | -------- | ---- | -------- | --------------------------------------------- | ---------------------------------------------------- |

- INDEX: `vehicle_type`
- INDEX: `created_by`

**SQL建表语句**：

```sql
-- =====================================================
-- 车辆表
-- =====================================================
CREATE TABLE `vehicles` (
  `id` INT(11) NOT NULL AUTO_INCREMENT COMMENT '车辆ID（主键）',
  `plate_number` VARCHAR(20) NOT NULL COMMENT '车牌号',
  `brand_model` VARCHAR(100) NOT NULL COMMENT '品牌型号',
  `vehicle_type` ENUM('gasoline', 'electric', 'hybrid') NOT NULL COMMENT '车辆类型',
  `initial_mileage` INT(11) NOT NULL DEFAULT 0 COMMENT '初始里程数（公里）',
  `is_active` TINYINT(1) NOT NULL DEFAULT 1 COMMENT '是否启用',
  `remarks` VARCHAR(200) DEFAULT NULL COMMENT '备注',
  `created_by` INT(11) NOT NULL COMMENT '创建者',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_plate_user` (`plate_number`, `created_by`),
  KEY `idx_vehicle_type` (`vehicle_type`),
  KEY `idx_created_by` (`created_by`),
  CONSTRAINT `fk_vehicle_creator` FOREIGN KEY (`created_by`) REFERENCES `users` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='车辆表';

-- =====================================================
-- 加油/充电记录表
-- =====================================================
CREATE TABLE `vehicle_fuel_records` (
  `id` INT(11) NOT NULL AUTO_INCREMENT COMMENT '记录ID（主键）',
  `vehicle_id` INT(11) NOT NULL COMMENT '车辆ID',
  `record_type` ENUM('fuel', 'charge') NOT NULL COMMENT '记录类型：加油/充电',
  `record_date` DATE NOT NULL COMMENT '日期',
  `record_time` TIME DEFAULT NULL COMMENT '时间',
  `amount` DECIMAL(10,2) NOT NULL COMMENT '金额（元）',
  `current_mileage` INT(11) NOT NULL COMMENT '当前里程数（公里）',
  `trip_mileage` INT(11) DEFAULT NULL COMMENT '本次行驶里程（公里）',
  `cost_per_km` DECIMAL(8,4) DEFAULT NULL COMMENT '每公里成本（元/km）',
  `remarks` VARCHAR(200) DEFAULT NULL COMMENT '备注',
  `created_by` INT(11) NOT NULL COMMENT '创建者',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_vehicle_id` (`vehicle_id`),
  KEY `idx_record_date` (`record_date`),
  KEY `idx_record_type` (`record_type`),
  KEY `idx_created_by` (`created_by`),
  CONSTRAINT `fk_fuel_record_vehicle` FOREIGN KEY (`vehicle_id`) REFERENCES `vehicles` (`id`) ON DELETE CASCADE,
  CONSTRAINT `fk_fuel_record_creator` FOREIGN KEY (`created_by`) REFERENCES `users` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='加油/充电记录表';
```

**接口地址**: `POST /api/vehicle/fuel`

```json
{
  "vehicle_id": 2,
  "record_type": "fuel",
  "record_date": "2026-04-16",
  "record_time": "09:00:00",
  "amount": 380.0,
  "current_mileage": 12580,
  "remarks": ""
}
```

**请求参数（充电）**:

```json
{
  "vehicle_id": 1,
  "record_type": "charge",
  "record_date": "2026-04-16",
  "record_time": "19:00:00",
  "amount": 28.5,
  "current_mileage": 8880,
  "remarks": "夜间谷电"
}
```

**后端处理逻辑**:

1. 接收基础数据（车辆ID、类型、日期、金额、里程数、备注）
2. 查询该车辆上一条记录的里程数
3. 计算：trip_mileage = current_mileage - 上一条里程
4. 计算：cost_per_km = amount ÷ trip_mileage
5. 如果 current_mileage < 上一条记录的里程数，返回警告但不阻止保存
6. 保存记录并返回完整数据（含自动计算的字段）

#### 2.3 更新记录

**接口地址**: `PUT /api/vehicle/fuel/:id`

**请求参数**: 同新增，所有字段可选

**注意**: 更新后需要重新计算后续记录的成本数据（如果里程数有变动）

#### 2.4 删除记录

**接口地址**: `DELETE /api/vehicle/fuel/:id`

**注意**: 删除后需要重新计算后续记录的成本数据

## 技术实现要点

### 1. 状态管理（Pinia Store）

**useVehicleStore**:

- vehicles: Vehicle[] - 车辆列表
- records: FuelRecord[] - 记录列表
- statistics: VehicleStatistics | null - 统计数据
- loading: boolean - 加载状态
- pagination: Pagination - 分页信息
- filters: VehicleFilters - 筛选条件

**主要 Actions**:

- fetchVehicles() - 获取车辆列表
- createVehicle(data) - 新增车辆
- updateVehicle(id, data) - 更新车辆
- deleteVehicle(id) - 删除车辆
- fetchRecords(filters, page, pageSize) - 获取记录列表
- createRecord(data) - 新增记录（含自动计算逻辑）
- updateRecord(id, data) - 更新记录
- deleteRecord(id) - 删除记录
- fetchStatistics(startDate, endDate) - 获取统计数据
- exportRecords(params) - 导出记录

### 2. 自动计算逻辑（前端+后端双重保障）

**前端实时计算**（提升用户体验）：

- 用户输入金额和里程 → 实时显示本次行驶里程和每公里成本
- 从本地缓存的上一条记录获取上次里程数
- 即时反馈，无需等待保存

**后端校验计算**（确保数据准确）：

- 保存时重新从数据库读取上一条记录
- 使用数据库中的准确数据进行计算
- 处理并发情况下的数据一致性

### 3. 数据一致性保障

**场景1：插入中间记录**

- 如果用户补录了之前的记录，后续记录的单位成本需要重新计算
- 方案：提供"重新计算"按钮，批量更新后续记录

**场景2：删除记录**

- 删除某条记录后，其下一条记录的"上次里程"会改变
- 方案：删除时提示是否重新计算后续记录

**场景3：修改里程数**

- 如果修改了某条记录的里程数，后续记录都需要重新计算
- 方案：保存时提示并自动更新后续记录

### 4. 性能优化

- **缓存策略**：统计数据缓存5分钟，避免频繁查询
- **懒加载**：图表数据按需加载
- **分页加载**：记录列表采用分页，默认每页20条

## 注意事项

1. **里程数连续性**：系统依赖里程数的连续性来计算成本，如果用户更换了车辆仪表盘或里程表归零，需要在车辆设置中说明
2. **混合动力车型**：对于插电混动车型，同时支持加油和充电两种记录类型
3. **数据精度**：金额保留2位小数，每公里成本保留4位小数
4. **隐私保护**：车牌号等敏感信息需要考虑脱敏展示（如：京A\*\*\*5）
5. **多用户支持**：每个用户独立管理自己的车辆数据，互不干扰
6. **极简原则**：本模块的核心价值在于"记录金额+里程→自动算成本"，避免过度设计导致使用复杂
